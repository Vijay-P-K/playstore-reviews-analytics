## 🚀 Project Overview

This project analyzes Google Play Store application data to uncover insights about app categories, pricing models (Free vs Paid), ratings, and user engagement trends.

The project simulates a real-world data analytics workflow including:

- Data cleaning using Python
- Data warehousing in Snowflake
- SQL-based transformations
- Interactive dashboard development in Looker Studio
- Version control using Git & GitHub

---

## 🎯 Business Objective

App marketplaces contain thousands of applications across multiple categories.  
This project aims to answer key business questions:

- Which app categories have the highest number of applications?
- What is the distribution of Free vs Paid apps across categories?
- Do Paid apps have higher ratings than Free apps?
- Which categories show strong market competition?

The insights help understand pricing strategies and market trends.

## 🔄 Data Pipeline Workflow

1. Raw CSV ingestion
2. Data cleaning & preprocessing (Python)
3. Upload cleaned data to Snowflake
4. SQL transformations & aggregations
5. Dashboard creation in Looker Studio
6. Insight generation

---

## 📊 Key Insights

- Free apps dominate across most categories.
- Paid apps are concentrated in niche segments like Business & Tools.
- Higher install count does not always correlate with higher ratings.
- Category competition varies significantly by app type.
